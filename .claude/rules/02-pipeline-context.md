# 프로젝트 맥락 형성 규칙

작업 시작 전 연관 파일을 모두 Read/Grep으로 확인한다.
파일명, 폴더명, 프레임워크 이름만 보고 구조를 확정하지 않는다.

## 저장소 구조 감지 원칙

- 언어와 프레임워크는 루트 의존성 매니페스트, 잠금 파일, 빌드 설정, 런타임 설정 파일을 읽어서 확인한다
- 앱 진입점은 실제 라우트, 명령, 서버, 페이지, 작업 스케줄 등록 파일에서 확인한다
- 프런트엔드, 백엔드, 작업자, 데이터 계층, 외부 서비스 모듈은 프로젝트마다 이름이 다르므로 실제 파일 기준으로만 분류한다
- 루트의 주요 디렉터리는 후보 경로일 뿐이며 실제 역할은 파일 내용으로 확정한다
- 배포 패키지에는 특정 프로젝트명, 특정 API URL, 특정 DB명, 개인 경로, 접속 계정을 남기지 않는다

## A. 화면·UI 작업

| 순서 | 확인 대상 | 확인 목적 |
|------|----------|----------|
| 1 | 실제 UI 진입점 | 라우팅, 페이지 등록, 컴포넌트 연결 |
| 2 | 공유 타입·스키마·뷰 모델 | 인라인 타입 및 중복 구조 방지 |
| 3 | API 호출 또는 데이터 로더 | 요청 경로, 응답 타입, 캐시 흐름 |
| 4 | 백엔드 라우트 또는 컨트롤러 | 실제 등록 여부와 응답 필드 |
| 5 | 서비스·도메인 로직 | 반환값과 UI 표시 필드 일치 |
| 6 | 저장소·DB·파일 계층 | 저장 필드와 조회 필드 일치 |
| 7 | 대상 UI 파일 | 현재 표시 항목, 상태, 계산식 보존 |

## B. 백엔드·API 작업

| 순서 | 확인 대상 | 확인 목적 |
|------|----------|----------|
| 1 | 앱 시작 파일과 라우트 등록 파일 | 활성 엔드포인트와 등록 순서 |
| 2 | 대상 라우트·컨트롤러 | 요청, 검증, 응답 구조 |
| 3 | 서비스·도메인 로직 | 실제 업무 흐름 |
| 4 | 데이터 접근 계층 | 쿼리, 모델, 저장소 호출 |
| 5 | 스키마·마이그레이션 | 필드, 타입, 인덱스 일치 |
| 6 | 환경 설정 로더 | env, ini, json, yaml 등 키 로딩 |
| 7 | 호출 클라이언트 | 프런트 또는 외부 호출 경로와 일치 |

## C. DB·마이그레이션 작업

| 순서 | 확인 대상 | 확인 목적 |
|------|----------|----------|
| 1 | 현재 스키마 정의 | 테이블, 컬럼, 타입, 인덱스 |
| 2 | 마이그레이션 실행기 | 순서, checksum, rollback 정책 |
| 3 | DB 연결 설정 | 환경변수, 설정 파일, 풀 구성 |
| 4 | 변경 테이블 사용처 | 서비스, 라우트, 작업자, UI 영향 |
| 5 | 타입·응답 스키마 | DB 필드와 외부 노출 필드 일치 |

## D. 외부 서비스·작업자·파이프라인 작업

| 순서 | 확인 대상 | 확인 목적 |
|------|----------|----------|
| 1 | 서비스 모듈 진입점 | 작업 시작과 생명주기 |
| 2 | 설정 로더 | 서비스별 env 또는 ini 키 |
| 3 | 외부 API 클라이언트 | URL 구성, 인증, 요청 파라미터 |
| 4 | 파이프라인 제어 파일 | 수집, 정제, 검증, 저장 순서 |
| 5 | 저장소·큐·파일 계층 | 중복 저장, 누락, 재시도 흐름 |
| 6 | 관측·로그 파일 | DB에 임의 로그 테이블 생성 금지 여부 |

## 공통 금지

- 실제 라우트 등록 파일 미확인 상태에서 엔드포인트 활성 여부 판단 금지
- 생성 산출물 또는 라우팅 매니페스트 미확인 상태에서 화면 연결 완료 보고 금지
- 외부 서비스 운영 API나 DB를 다른 모듈에서 직접 호출하는 구조 금지
- 연관 파일 읽기 전 추측으로 타입·필드 일치 판단 금지
- 배포 파일에 개인 API 값, DB 접속값, 절대 경로, 특정 프로젝트명을 유지 금지

## 맥락 형성 완료 기준

- [ ] 실제 스택과 폴더 구조를 파일에서 확인했다
- [ ] 작업 유형별 순서표의 연관 파일을 확인했다
- [ ] 요청 진입점부터 응답, 저장, 표시까지 흐름을 설명할 수 있다
- [ ] 타입·스키마·DB 필드가 일치함을 실제 코드에서 확인했다
- [ ] 외부 API와 서비스 경계가 설정 파일과 실제 클라이언트 코드 기준으로 확인됐다
- [ ] 왜/어떻게/무엇을/어떤 식으로 4가지 질문에 답할 수 있다
