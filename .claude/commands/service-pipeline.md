이 명령은 배포 대상 프로젝트의 서비스형 모듈 작업을 실제 파일과 실제 흐름에 고정한다.
서비스 디렉터리명, 언어, 프레임워크를 고정하지 않는다.

## 실행 환경 고정

- 환경변수 기준 파일은 실제 설정 로더에서 확인한다.
- OS 전역 env, 추정값, 과거 세션 값을 기준으로 판단하지 않는다.
- 비밀값은 응답에 재출력하지 않는다.

## 절대 금지 실행

사용자가 명시적으로 허용하기 전까지 다음을 실행하지 않는다.

- 서비스 서버 기동
- 작업자, 큐 컨슈머, 스케줄러 기동
- 외부 API 실제 호출
- 에러 확인 목적의 런타임 실행

검증은 다음 방법으로 수행한다.

- 파일 전체 읽기
- 호출부, 반환값, 타입·스키마, 파이프라인 흐름 정적 추적
- 설정 로더와 외부 API 클라이언트 대조

## 파일 읽기 순서

1. 서비스 진입점
2. 설정 로더
3. 외부 API 클라이언트
4. 파이프라인 컨트롤러 또는 작업 오케스트레이터
5. 수집, 정규화, 검증, 중복 처리 모듈
6. 저장소, 큐, 파일 저장, DB 반영 모듈
7. 재시도, 추적, 로그, 관측 모듈
8. 상위 앱에서 서비스를 호출하는 경계 파일

파일을 읽은 후 각 파일 역할을 실제 코드 기준으로 정리한다.

## 중앙 통제 흐름 보존

다음 흐름이 실제로 연결되는지 확인한다.

```
진입점
  → 설정 로더
  → 외부 API 클라이언트 또는 큐 입력
  → 수집/정규화/검증
  → 중복 처리
  → 저장소/큐/파일/DB 반영
  → 재시도/추적/로그
```

## DB와 로그 처리 규칙

- 서비스 전용 DB가 있으면 다른 모듈에서 직접 쓰지 않는다.
- 로그용 DB 테이블을 임의로 추가하지 않는다.
- 로그는 프로젝트의 기존 로그 정책과 저장 위치를 따른다.
- 새 저장 구조는 실제 마이그레이션과 호출부를 확인한 뒤 추가한다.

## 외부 API 규칙

- 기본 URL은 설정 키 이름만 보고한다.
- 실제 URL, 서비스 키, 토큰, 계정, 비밀번호는 재출력하지 않는다.
- 오퍼레이션명, 요청 파라미터, 응답 필드는 공식 문서와 실제 클라이언트 코드를 대조한다.

## 완료 전 자체 점검

- [ ] 서비스 디렉터리와 진입점을 실제 파일에서 확인했다
- [ ] 설정 키 존재 여부를 확인했다. 비밀값은 재출력하지 않았다
- [ ] 외부 API 문서와 클라이언트 코드를 대조했다
- [ ] 진입점 → 설정 → 외부 호출 → 처리 → 저장 → 추적 흐름이 이어지는지 확인했다
- [ ] 역할 경계를 침범하는 수정이 없는지 확인했다
- [ ] 금지된 서버 또는 작업자 기동 명령을 실행하지 않았다
- [ ] 확인한 파일과 흐름만 완료로 보고했다
