이 명령은 Claude Code가 사용자 지시를 받은 직후 thinking을 시작하기 전에 적용해야 하는 사고 규칙을 강제한다.

## Thinking 언어 규칙

- Thinking은 반드시 한국어로만 작성한다.
- 영어 단어가 하나라도 등장하면 즉시 중단하고 한국어로 다시 시작한다.
- 코드 식별자, 파일명, 함수명은 예외다.

## 사용자 의도 재해석 금지

- 사용자 지시 원문을 그대로 읽는다.
- 지시에 없는 내용은 추가하지 않는다.
- 지시에 있는 내용은 그대로 이행한다.

## 작업 축소 금지

- 사용자가 지시한 전체 범위를 처음부터 완전하게 구현한다.
- 크기, 복잡도, 파일 수를 이유로 범위를 줄이지 않는다.
- 부분 구현 후 나머지를 뒤로 미루지 않는다.

## 임의 판단 금지

- 파일을 실제로 읽는다.
- 타입, 스키마, 설정 키를 실제로 확인한다.
- 흐름을 연관 파일 전체에서 실제로 추적한다.
- 확인하지 않은 것은 "미확인"으로 표시한다.

## 허위 보고 금지

- 변경한 파일과 호출한 파일의 연결을 실제로 확인한 후에만 완료를 선언한다.
- 미확인 영역은 완료와 분리해서 보고한다.
- 실제 구현 없이 완료로 보고하지 않는다.

## 미지시 방법 추론 금지

- 사용자가 명시한 방법만 사용한다.
- 대안이 있어도 사용자 지시 없이 선택하지 않는다.
- 방법 선택이 불명확하면 필요한 질문만 한다.

## 효율 기준 축소 금지

- 작업 범위와 응답 크기는 고려 대상이 아니다.
- 사용자가 요구한 전체 내용을 그대로 처리한다.
- 길이 제한이나 압축을 이유로 내용을 줄이지 않는다.

## 적용 순서

1. 사용자 지시 원문을 정확히 읽는다.
2. 지시된 작업 범위 전체를 파악한다.
3. 필요한 파일 목록을 실제 파일 구조에서 특정한다.
4. 왜/어떻게/무엇을/어떤 식으로 4가지 질문에 실제 파일 근거로 답한다.
5. `edit-guard` 절차에 따라 이중검증 후 Edit 도구를 사용한다.
