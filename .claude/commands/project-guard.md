이 명령은 배포 대상 프로젝트의 작업을 실제 파일, 실제 설정 로딩 경로, 실제 데이터 흐름에 고정한다.
언어, 프레임워크, 폴더 구조를 고정하지 않는다.

## 실행 환경 고정

텍스트 인코딩이 필요한 명령에는 다음 값을 적용한다.

```bash
export LANG=ko_KR.UTF-8
export LC_ALL=ko_KR.UTF-8
export PYTHONUTF8=1
export PYTHONIOENCODING=UTF-8
```

## 절대 금지 실행

사용자가 명시적으로 허용하기 전까지 다음을 실행하지 않는다.

- 개발 서버 기동 명령
- 프로덕션 빌드 명령
- 프리뷰 서버 명령
- 서버 재시작 명령
- 외부 서비스 실제 호출 명령

검증은 다음 방법으로 수행한다.

- 정적 독해
- 파일 간 호출 흐름 확인
- 타입·스키마·설정 정의 확인
- 라우트 또는 진입점 등록 확인
- 마이그레이션과 저장소 정의 확인

## 시작 순서

1. `AGENTS.md`, `CLAUDE.md`, `rules/`에서 현재 작업 규칙을 확인한다.
2. 루트 설정 파일을 읽어 실제 언어와 프레임워크를 확인한다.
3. env, ini, json, yaml, toml 설정 파일의 로딩 경로를 확인한다.
4. 비밀값은 응답에 재출력하지 않는다.
5. 외부 API 작업이면 공식 문서와 실제 클라이언트 코드를 대조한다.

## 코드 추적 순서

1. 앱 또는 작업 진입점
2. 라우트, 컨트롤러, 명령, 작업자 등록 파일
3. 서비스, 도메인 로직, 유스케이스
4. 저장소, 모델, DB, 파일, 큐 계층
5. 타입, DTO, 검증 스키마, 응답 스키마
6. UI 또는 외부 호출 클라이언트

## 구현 원칙

- 가짜 구현, 더미 데이터, 임시 처리, 주석 처리 우회, 삭제로 문제를 숨기는 수정 금지
- 타입, 스키마, 공통 로직은 책임 위치에 중앙화한다
- 기존 사용자 변경은 되돌리지 않는다
- 한국어 파일은 국소 편집하고 UTF-8로 다시 읽어 깨짐을 확인한다
- 기존 파일은 Edit 도구로만 수정한다
- 배포 파일에는 실제 API URL, DB 접속값, 개인 절대 경로, 특정 프로젝트명을 남기지 않는다

## 완료 전 자체 점검

- [ ] 실제 프로젝트 스택을 파일에서 확인했다
- [ ] 설정 로딩 경로와 키 이름을 확인했다
- [ ] 비밀값을 재출력하지 않았다
- [ ] 진입점, 서비스, 저장소, 타입 또는 화면 흐름을 앞뒤로 추적했다
- [ ] 금지된 빌드 또는 개발 서버를 실행하지 않았다
- [ ] 배포 파일의 프로젝트 고유 값이 자리표시자로 교체됐다
- [ ] 미확인 영역은 완료가 아닌 미확인으로 분리해서 보고했다
