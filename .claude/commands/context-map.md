이 명령은 배포 대상 프로젝트에서 작업을 시작하기 전에 폴더 트리별 연관 파일을 실제 구조 기준으로 읽게 한다.
언어, 프레임워크, 폴더명은 고정하지 않는다.

## 저장소 구조 감지

- 루트 설정 파일: 의존성 매니페스트, 잠금 파일, 빌드 설정, 런타임 설정
- UI 후보: `src`, `app`, `pages`, `components`, `resources`, `templates`, `views`
- 백엔드 후보: `server`, `app`, `api`, `routes`, `controllers`, `services`
- DB 후보: `database`, `migrations`, `models`, `entities`, `repositories`, `schema`
- 서비스 후보: `services`, `workers`, `jobs`, `queues`, `tasks`, `pipelines`, `packages`

## 작업 유형별 필수 참조 순서

### A. UI 작업

| 순서 | 확인 대상 | 확인 목적 |
|------|----------|----------|
| 1 | UI 진입점과 라우팅 등록 | 화면 연결 확인 |
| 2 | 공유 타입, 스키마, 뷰 모델 | 중복 타입 방지 |
| 3 | API 호출 또는 데이터 로더 | 요청·응답 필드 확인 |
| 4 | 백엔드 라우트 또는 컨트롤러 | 실제 등록 여부 확인 |
| 5 | 서비스와 저장소 | 반환값과 저장 필드 확인 |
| 6 | 대상 UI 파일 | 기존 표시 항목과 상태 보존 |

### B. 백엔드 작업

| 순서 | 확인 대상 | 확인 목적 |
|------|----------|----------|
| 1 | 앱 시작 파일과 라우트 등록 파일 | 활성 엔드포인트 확인 |
| 2 | 대상 라우트, 컨트롤러, 핸들러 | 요청·응답 구조 확인 |
| 3 | 서비스 또는 도메인 로직 | 실제 업무 흐름 확인 |
| 4 | 저장소, 모델, DB 계층 | 저장·조회 필드 확인 |
| 5 | 타입, DTO, 검증 스키마 | 외부 노출 필드 일치 확인 |

### C. DB 작업

| 순서 | 확인 대상 | 확인 목적 |
|------|----------|----------|
| 1 | 현재 스키마와 마이그레이션 목록 | 테이블·컬럼·타입 확인 |
| 2 | 마이그레이션 실행 방식 | 순서·checksum·rollback 확인 |
| 3 | DB 연결 설정 | env 또는 설정 로더 확인 |
| 4 | 변경 테이블 사용처 | 서비스·라우트·UI 영향 확인 |
| 5 | 응답 타입과 외부 스키마 | 필드 일치 확인 |

### D. 서비스·작업자·외부 연동 작업

| 순서 | 확인 대상 | 확인 목적 |
|------|----------|----------|
| 1 | 서비스 진입점 | 작업 시작 흐름 확인 |
| 2 | 설정 로더 | 키 이름과 값 로딩 위치 확인 |
| 3 | 외부 API 클라이언트 | URL 구성과 인증 처리 확인 |
| 4 | 처리 파이프라인 | 수집·정규화·검증·저장 순서 |
| 5 | 로그·재시도·추적 | 관측과 실패 처리 확인 |

## 공통 금지 사항

- 실제 등록 파일 미확인 상태에서 라우트 활성 여부 판단 금지
- 생성 산출물 또는 라우팅 매니페스트 미확인 상태에서 연결 완료 보고 금지
- 외부 서비스 운영 API나 DB를 다른 모듈에서 직접 호출 금지
- 연관 파일 읽기 전 추측으로 타입·필드 일치 판단 금지
- 배포 파일에 실제 API 값, DB 접속값, 개인 절대 경로 유지 금지

## 맥락 형성 완료 기준

- [ ] 실제 언어와 프레임워크를 파일에서 확인했다
- [ ] 작업 유형에 해당하는 파일을 모두 확인했다
- [ ] 요청 또는 작업 진입점부터 응답, 저장, 표시까지 흐름을 확인했다
- [ ] 타입·스키마·DB 필드가 일치함을 확인했다
- [ ] 외부 API와 서비스 경계를 실제 클라이언트 코드에서 확인했다
